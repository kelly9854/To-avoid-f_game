# To-avoid-f_game
F를 피하는 방법 가상 시뮬레이션 게임


주제선정 이유
많은 주제들 중 프로젝트를 하는데 있어서 팀원들 전체가 흥미를 가지고 접근할 수 있는 주제가 우리 근처에 있는 성적과 학점일 것 같아서 선정하게 되었습니다. 학점을 구성하는 여러 가지 요소들을 객체개념을 이용해 나타내고, 추가로 ‘행복도’라는 개념을 이용해 실제 우리들의 모습과 비슷한 프로그램을 만들어보고 싶었습니다.


구성
1. 행복 Class: 
학생의 행복지수가 설정됩니다(0~100). 초기값은 50으로 설정.
공부능력과 상반되는 class로 출석을 하지않거나(출튀, 결석) 공부를 하지 않을 때 행복도가 상승합니다. 반대로 출석을 하거나 공부를 할 때 행복도가 감소합니다. 최종성적과 행복도에 따라 엔딩이 달라집니다. 학점을 위해 너무 공부만 해서 행복도가 0이되면 학생은 참지못하고 학기 도중에 자퇴를 하게 됩니다. 

2. 공부능력 Class: 
학생의 공부능력이 설정됩니다(0~100) 초기값은 50으로 설정.
출석을 하거나 공부를 하면 공부능력이 상승합니다. 반대로 출석을 하지 않거나 공부를 하지 않을 경우 공부능력이 감소합니다. 공부학기 중 쌓은 공부능력에 따라 마지막 기말고사에서 시험성적이 갈립니다.
행복 클래스와 공부능력 클래스는 상속 개념을 이용하여 이벤트 발생 시 같은 수치만큼 증가/감소 하게 됩니다.

3. 출석 Class: 
매주 출석체크를 합니다. Week 1 ~ Week 7 매주차 출석유무가 결정됩니다.
Week 8는 기말고사 주간으로 출석체크는 하지 않습니다.
출석여부를 입력받습니다. (출석 / 출튀 / 결석)
출석: 공부능력이 상승하고 행복도가 감소합니다.
출튀: 공부능력이 약간 감소하고 행복도가 약간 증가합니다. (랜덤확률로 결석처리 됩니다.)
결석: 공부능력이 감소하고 행복도가 증가합니다.
7주차중 3번 이상 결석처리되면 제적처리 됩니다. (자동으로 F학점을 받습니다.)



4. 매주테스트 Class:
매주 쪽지시험이 진행됩니다. (총 6번)사용자에게 입력을 받습니다.(공부한다/안한다)
공부한다를 선택하면 공부능력이 상승 / 행복도가 감소합니다. (시험점수: 6~10점 랜덤)
안한다를 선택하면 행복도가 상승 / 공부능력이 감소합니다. (시험점수 0~5점 랜덤)
출석 class에서 결석을 선택하면 매주테스트 점수는 0입니다.
1주차, 기말고사에는 매주테스트를 실시하지 않습니다.

5. 기말고사 Class: 
Week8에 기말고사를 치릅니다. 지금까지 쌓아온 공부능력에 따라 기말고사 성적이 결정됩니다.(0~100).

6. 엔딩 Class: 
메인클래스 역할을 하여 각 개체들을 연결하며, 지금까지의 매주테스트 + 기말고사를 종합해서 
최종학점을 결정합니다(A ~ F). 최종학점과 행복도에 따라서 다양한 엔딩이 나타납니다.
엔딩에 맞게 파일입출력을 통해 엔딩을 화면에 출력한 뒤 파일로 저장합니다.
